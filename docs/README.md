# OctoAcme Project Management Docs

This README provides a summary of project management processes and links to all process docs used by OctoAcme.

## Summary of Project Management Processes

OctoAcme uses an iterative, customer-first approach with clear roles, regular milestones, risk management, and continuous improvement:

- **Initiation**: Define goals, success metrics, stakeholders, & risks
- **Planning**: Decompose work, estimate, identify milestones/dependencies
- **Execution**: Daily standups, PR/branch review, QA/testing, reporting
- **Releases**: Standardized pre-release, deployment, and rollback procedures
- **Retrospective**: Capture learnings and action items for continuous improvement

## Using These Docs as Copilot Context

These documents are designed to be added to Copilot Spaces as context. When you attach them to a Copilot Space:
- You can reference specific processes or roles by name
- Copilot can provide role-specific guidance (e.g., "What should I do as a Scrum Master in this scenario?")
- All decisions and escalation paths are available for consultation

Add the entire `docs/` folder to your Copilot Space for comprehensive project management context.
